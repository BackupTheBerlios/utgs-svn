Starting Examples

To run examples, you'll need following steps:

  • Download (or build) KooLiXP executable,
  • Download (or checkout) example, and extract to examples directory,
  • Download azure and graphite theme and extract to themes directory,
  • Run command in example directory: KooLixP.exe start_azure.xlp (this one is for movie player example with azure theme)

Build Prerequisites

Before you build UTGS you'll need following steps:

  • Checkout the latest version of library: svn checkout svn://svn.berlios.de/utgs/trunk
  • Download Boost.Org library, and set BOOST_ROOT environment variable,
  • Download CryptoPP library, and set CRYPTOPP_ROOT environment variable,
  • Download Ogg/Vorbis library, and set OGGVORBIS_ROOT environment variable.
  • One may also need to download Direct X SDK at least version 8.1, and set DXSDK environment variable.

Project Description

Mission

There are many libraries doing similar things like UTGS does. But I could rarely find anything friendly to developers. Those Other libraries frequently introduce very unclear interfaces. One will surely get lost in those plenty methods per class. Here in UTGS I introduce very simple and short interfaces (See C++ concepts overview...). In addition it gives scripting language, that is easy to learn, and able to control the user interface. Using this language you may even write your whole application. (See KooLiXP overview...).

Project Scope

Free or Commercial Multimedia Applications & Games.

Overview

To get started read following documents:

  • About UTGS
  • Scripting language overview
  • C++ components overview
To see documents listed above you'll need XML/XSLT capable browser.

Virtues

  • BSD License (allows commercial projects),
  • X-platform project via multiple layers of abstraction (*),
  • Scripting language, easy to learn, with many bindings to components of the library,
  • Extensible thru external modules loaded at run-time,
  • Tested against comercial game projects,

(*) Currently supported: Win32, DirectDraw7, OpenGL, DirectShow, Ogg/Vorbis

Features

  • Encryption Capable Virtual Filesystem (pack and encrypt all application files into one or several files),
  • GUI User Interface Library (create and place groups of buttons and other even those complicated gadgets),
  • XML Resource Library (create GUI in similar way like HTML pages are desigend),
  • C++ Utility Templates (bind actions with GUI elements and more...),
  • DirectShow streaming thru OpenGL textures synchronized with GUI rendering.

Assumptions

  • Just use this library, since it is working nicely,
  • Code is written in C++ language,
  • There are several bugs to fix,
  • New components might be added, such as 3d graphics and physics (*)
(*) 3D graphics and physics engine is to be added. We could just reuse some free existing 3D engine, but I couldn't find any just good enough.
